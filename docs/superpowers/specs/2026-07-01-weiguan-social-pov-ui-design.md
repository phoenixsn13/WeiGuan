# 围观 Social POV UI 设计稿

## 目标

本轮 UI/UX 整改把围观从“后端调试页”拉回原始产品设定：用户看到的是熟悉的社交媒体体验，而不是仿真工作台。专业推演能力保留在产品底层，前台表达为“我发了一条内容，大家如何围观、发酵、分化、收束”。

禁止把以下概念作为普通用户主界面语言：`agent`、`OASIS`、`仿真`、`工作台`、`trace`、`simulation`、`dashboard`。这些只允许出现在开发文档、调试日志或隐藏的高级配置里。

- 时间线图：`发酵时间线 + 创作者流量洞察` 是正确方向；第一批实现时间线骨架和代表评论，不一次性堆满指标。

## 产品视角

### 历史推演

进入 Live 之前，用户应该能查看历史推演，而不是只能看当前正在跑的这一次。用户心智是“我之前围观过哪些内容，点开继续看评论区或复盘”。

- 首页或导航中提供 `历史记录`。
- 历史卡片展示：seed 内容摘要、圈子/平台、步数、评论数、转发数、状态、创建时间。
- 每条历史提供两个入口：`看评论区` 与 `看回放`。
- 第一批实现当前服务生命周期内的历史列表；跨服务重启的磁盘持久化作为第二批后端存储任务。

### 我看到的

默认 Live 视角。用户心智是“我发出去以后，大家怎么回我”。界面像一条微博/X 详情页：

- 顶部：返回、`我看到的`、轮次进度、运行状态。
- 主体：seed 帖固定在上方。
- 分区：`评论`、`转发`、`通知`。
- 评论：在固定高度容器内滚动，不把页面无限撑长。
- 控制：底部悬浮播放条，属于围观壳，不污染社交皮肤。

### TA 看到的

用户点击评论者后可以切换到这个人的视角。用户心智是“这个人当时看到了什么，所以才这么说”。

- 顶部出现 `正在从 @handle 的视角看`。
- 有明确的 `回到我看到的`。
- seed 帖仍以社交 feed 方式出现。
- TA 自己的评论/动作高亮。
- 只展示 TA 合理可能注意到的内容，不暴露全局推演日志。
- 追问入口仍然可用，但必须接地到同一次 run 的真实评论和动作。

### 围观回放

复盘页不是数据后台，而是“社交发酵时间线”。用户心智是“这条内容怎么一步步被理解、质疑、转发和收束”。

默认结构：

1. 第 1 波：第一批人看到。
2. 第 2 波：质疑点出现。
3. 第 3 波：立场分化。
4. 第 4 波：结论。

每一波展示：

- 代表评论。
- 评论/点赞/转发/参与人数摘要。
- 情绪标签。
- 关键争议词。

右侧可以放创作者流量洞察，但第一批只做紧凑摘要：评论数、转发数、整体情绪、风险点、建议。

## 布局规范

### Live 桌面端

```text
┌───────────────────────────────────────────────┐
│ 围观 · 我看到的                  第 4/10 步     │
├───────────────┬───────────────────┬───────────┤
│ 轻导航/状态     │ seed 帖固定          │ 通知摘要    │
│ 当前推演信息     ├───────────────────┤           │
│               │ 评论/转发/通知 tabs   │           │
│               ├───────────────────┤           │
│               │ 内部滚动评论窗口       │           │
├───────────────┴───────────────────┴───────────┤
│             底部悬浮播放控制条                  │
└───────────────────────────────────────────────┘
```

### Live 移动端

- 单列。
- 顶部固定导航。
- seed 帖固定或 sticky 在评论区上方。
- 评论区占据主要高度。
- 通知合并到 tabs。
- 底部控制条固定在安全区上方。

## 组件计划

新增或改造前端组件：

- `LiveScreen`：页面骨架、视角状态、底部控制条、路由跳转。
- `XFeed`：社交皮肤入口，接收 view model，不直接读 snapshot。
- `XFeedFrame`：固定高度社交 App 容器。
- `XThreadTabs`：评论/转发/通知分区。
- `XScrollableThread`：内部滚动评论窗口。
- `XNotificationRail`：桌面右侧通知摘要，移动端收进 tab。
- `PerspectiveBanner`：显示 `我看到的` 或 `正在从 @handle 的视角看`。
- `ActorPerspectiveView`：第一批可先复用 `XFeed` 数据结构，高亮选中 actor 的动作。
- `ReplayTimeline`：复盘页的发酵时间线。
- `ReplayWaveCard`：一波讨论阶段卡片。

## 配色与字体

颜色遵循原设计两个寄存器：

- 社交皮肤区：近白背景 `#FFFFFF`、正文 `#0F172A`、次级文字 `#64748B`、分割线 `#E2E8F0`、互动蓝 `#2563EB`。
- 围观壳：深墨 `#0F172A` / `#14140F`、暖琥珀 `#E8A13A`、冷靛蓝 `#2C4A7C`。
- 情绪：正向 `#3E9B6E`、负向 `#C4553B`、中立 `#8A8578`。

字体：

- UI 正文：系统无衬线，优先 `-apple-system, BlinkMacSystemFont, "Segoe UI", "Noto Sans SC", sans-serif`。
- 数字：`tabular-nums`。
- 不使用 emoji 作为结构图标；按钮和导航使用 SVG 或文本。

## 交互规则

- 评论新增时淡入，尊重 `prefers-reduced-motion`。
- 评论区默认跟随底部；用户向上滚动后不强行跳底。
- 所有点击目标至少 44px 高。
- `看结果` 只在 run 完成后可用。
- 点击评论作者：
  - 打开追问抽屉；
  - 同时允许切换到 TA 视角；
  - 对未参与 seed 的 actor 不显示追问入口。

## 第一批实现范围

第一批只解决当前最影响体验的问题：

1. 历史推演列表，能进入某次 run 的评论区或回放。
2. Live 页固定视窗，不再无限撑长页面。
3. seed 帖固定，评论在内部滚动。
4. 增加顶部社交导航、视角文案和进度。
5. 增加评论/通知 tabs，使用现有 `thread` 和 `notifications`。
6. 点击评论者后出现 `正在从 @handle 的视角看` 状态，并高亮该 actor 的评论。
7. Retro 页改为“围观回放 / 发酵时间线”的表达，不再只是三条进度条。
8. Playwright 截桌面和移动端图，写 manual review。

## 验收标准

- Live 页面在大量评论下整体页面高度稳定，评论区内部滚动。
- 首屏能一眼识别为社交媒体内容页，不像后台工作台。
- UI 中不出现 `agent/OASIS/仿真/工作台`。
- 历史记录能列出已创建 run，并从历史进入 Live 与 Retro。
- 第一人称、某人视角、回放三个入口/状态的文案清楚。
- 复盘页以 timeline 表达发酵过程。
- `npm test`、`npx tsc -b` 通过。
- Playwright 产出 desktop/mobile 截图并落盘到 manual 文档。
