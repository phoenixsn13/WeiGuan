# 围观平台风味评估 Rubric

本文档用于审核者或 LLM 对 `FlavorDigest` 做可复现的主观评估。`FlavorDigest` 是事件日志/快照的确定性投影，不是 LLM 产物。

## 输入

- `FlavorDigest.platforms[*].phases[*].representative_utterances`
- `persona_mix`
- `spread_shape`
- `cross_platform_notes`

评估时只引用摘要中的代表原话，不回看未进入摘要的原始日志。

## 五个维度

1. **语域**
   - 判断口语、正式表达、梗密度、反讽密度是否贴近对应平台。
   - 微博更偏短句、情绪化、话题标签和围观语气。
   - Reddit 更偏解释、追问、楼中楼引用和投票语境。
   - X 更偏短促、quote-dunk、观点压缩。

2. **平台记号**
   - 微博：控评、转发链、`#话题#`、大 V 带节奏。
   - Reddit：楼中楼、引用对喷、subreddit/投票语气。
   - X：quote、短促转述、转推式嘲讽。

3. **persona 行为一致性**
   - KOL 是否像在带节奏。
   - Verified/大 V 是否更像权威解释或立场定调。
   - 普通人是否表现为跟风、质疑、补充、玩梗。

4. **发酵自然度**
   - 结合 `phases` 与 `spread_shape` 判断扩散是否像真实发酵。
   - 优先识别：单源爆发、多点扩散、链式转发、零散。
   - 警惕机械四波脉冲、阶段文本重复、同质化严重。

5. **跨平台转述变味**
   - 观察 `cross_platform_notes` 与各平台代表原话。
   - 同一话题经桥接到另一平台后，是否换成该平台更自然的说法。

## 输出格式

每个平台输出：

- 评级：`很像` / `基本像` / `有点像` / `不像`
- 主要依据：引用 2-5 条代表原话
- 风味短评：指出最像和最不像的地方
- 改进建议：只基于摘要内容提出，不引入外部推测

整体输出：

- 跨平台转述是否自然
- 发酵曲线是否像真实扩散
- persona 是否可辨

## 注意

- 不根据接口名、代码名、日志名评分。
- 不把 perf 指标当作平台风味依据。
- 不要求每个平台都出现所有平台记号，重点看是否符合当前话题和阶段。
