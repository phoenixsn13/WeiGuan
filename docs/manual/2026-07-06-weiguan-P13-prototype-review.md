## 采纳的设计约束

1. 顶层导航统一为深色壳，主工作区使用浅灰背景和白色内容面，强调色使用琥珀色。
2. 信息架构使用“发起 / 世界 / 历史”，用户看到的是社交媒体现场、历史和回放，不出现技术心智词。
3. 多平台现场使用平台列，桥接信息进入文档流；无桥接事件时不占右侧整列。
4. 历史页以发起记录为主列表，热榜按同内容聚合，并显示“发起 N 次”。
5. 复盘页采用上下文栏、发酵时间线、洞察栏三段结构；移动端按同样顺序堆叠。
6. 发起页右栏必须有内容：成本预估、身份摘要、安全提示和主 CTA；身份选择使用可搜索/可折叠列表，不无限铺开。

## 自审结果

### §7.1 调性

- 通过：页面靠近社交媒体产品，主表达是微博/Reddit 现场、评论区、热榜、回放，而不是“推演工作台”。
- 通过：桌面和移动都保留同一套深色壳、白色信息流、琥珀强调色。
- 待实现时修正：发起复盘原型左侧出现了类似终端的视觉块。这不是合格产品元素，T6 实现时必须替换为平台角标、发起摘要或内容摘要，禁止复现 debug/terminal 视觉。

### §7.2 两条硬规则

- 配色：实现只能使用 `design/tokens.ts` 的语义色。原型中的深色壳、白色面、琥珀强调、绿色/红色情绪条都要落到 token，禁止直接使用 Tailwind 默认色阶补色。
- 显示名：原型不含裸长 ID。实现必须延续 P13-T1 的 `author_display_name -> persons join -> 围观者·尾4位` 解析链，页面文本不得命中 `/[0-9a-f]{12,}/`。

### §7.3 心智词表

- 采纳词：世界、身份、平台、现场、发酵、传播、桥接、拍。
- 禁用词：agent、OASIS、仿真、工作台、后端、模型、步。
- 发现：部分原型仍有“Web”等平台来源文案；实现时可以保留“来自 Web”作为社交来源，但不能暴露工程词。

## 实现计划摘录

- T3 多平台现场：保留状态徽标、平台列、桥接路径、移动平台切换；去掉 absolute 桥接线。
- T4 历史和世界总览：按发起记录统一入口，热榜按内容聚合。
- T5 回放/复盘减重：回放显示“回放 · 共 n 条评论”，不再出现“实时互动”；时间单位使用“拍”。
- T6 发起复盘：抽 `RunAnalysisPanel`，多平台复盘使用平台 tab、桥接摘要和风味摘要。
- T7 发起页：收紧首屏，CTA 常驻可见，身份选择从长列表改成搜索/折叠选择器。
