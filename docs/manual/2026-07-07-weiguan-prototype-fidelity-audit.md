# 围观终板原型高保真审查

日期：2026-07-07  
审查范围：对照 P13/P14 终板原型、最新 manual 截图和当前实现状态，重新判定桌面版是否达到高保真。移动端只记录明显偏离，不作为本轮主修范围。

## 审查方法

本轮不再用“功能可达”替代“高保真”。每个页面按以下问题逐项判断：

1. 原型里是否有明确结构、比例、图标、密度和文案。
2. 当前实现是否照着做。
3. 没照着做时，是否存在明确技术难点、数据契约限制或产品收益。
4. 若没有正当理由，直接标为实现偏离，不再描述为“轻微差异”。

高保真判断标准：

- **达标**：布局骨架、首屏比例、信息层级、图标风格、按钮层级、卡片密度和业务语义均接近原型。
- **部分达标**：核心心智成立，但一个以上主区域结构或密度偏离。
- **未达标**：页面能用，但首屏骨架、主要信息结构或视觉语言与终板原型不是同一套。

## 总体结论

当前桌面版不能称为 P13/P14 终板原型的高保真实现。  
更准确的现状是：后端契约和主要页面入口已接通，部分页面有接近原型的方向，但 UI 实现仍大量停留在“功能拼装 / 后台表单 / 管理列表”的层级。

这不是单个颜色、按钮或 iconfont 能补齐的问题。图标确实需要统一，但更大的差距是：

- 容器宽度、首屏比例和左右栏权重没有按原型锁定。
- 页面组件缺少原型中的业务分组方式，导致同样的数据被展示成管理页。
- 图标和平台标识使用文字符号或首字母替代，降低产品完成度。
- 多处说明文案、按钮层级和控件密度偏后台工具，而不是社交产品。
- 当前 manual 中“结构已对齐”“轻微差异”等表述过宽，需要修正为偏离清单。

## 逐页审查

### 1. 发起页 `/compose`

结论：**未达高保真**。

原型特征：

- 桌面首屏是 1280 左右的集中画布，左主栏约 2/3，右发起栈约 1/3。
- 左主栏只承载正文、世界、新建/继续世界、发布平台；右栏承载身份、讨论轮次、费用和唯一 CTA。
- 世界模块是同一个卡片内的左右双面板，不是两个均等表单块。
- 平台选择有品牌图标：微博、Reddit、多平台并发。
- 右栏身份卡是摘要式选择器，不是大面积 persona/mode 配置面板。

当前偏离：

- 页面整体宽度、背景色和顶部留白与原型明显不同；当前视觉更窄、更暖、更像表单后台。
- 右栏身份区域展开了普通人/大V/KOL、新身份/继续身份等全量控件，压过了“发帖身份摘要”的产品心智。
- 世界选区虽然有双栏，但比例、密度、列表样式、搜索框位置和原型差异较大。
- 发布平台使用 checkbox 和文字卡，缺少原型中的平台图标与选中角标。
- 图标仍有原生符号痕迹，如 `◎`、`⌁`、`◈`、`◌`，没有形成统一 iconfont/SVG 系统。

没照做的正当理由：

- 保留更多身份配置能力有产品价值，但不应破坏原型首屏。合理做法是折叠到“更换身份/高级设置”中，而不是铺开。
- 真实世界列表比原型样例更多，这是数据差异；但不解释布局、图标和密度偏离。

整改级别：**P0 视觉骨架重构**。

### 2. 世界总览 `/worlds`

结论：**未达高保真**。

原型特征：

- 左侧固定热榜卡，右侧是少量世界卡片信息流。
- 世界卡强调主身份、最近内容、状态、身份数、影响力、平台数、运行次数和三类入口按钮。
- 首屏可以扫到 3-4 个高质量世界，不是无限管理列表。
- 世界图标是彩色语义图标，而不是首字母圆点。

当前偏离：

- 当前更像“所有世界管理列表”，大量普通人/空世界铺满全页。
- 世界卡片高度、按钮位置、统计排布和原型不同；没有原型里的主身份关系和清晰状态列。
- 热榜区域尺寸和原型弱化，不能承担左栏导航/发现角色。
- 世界名中仍出现“普通人·hash 的世界”这类系统生成痕迹，削弱产品感。

没照做的正当理由：

- 真实数据里确实存在大量空世界，需要产品层过滤、归档或折叠；这不是保留长列表的理由。
- 若设计要支持 100+ 世界，应在原型语言内引入分页/搜索/筛选，而不是无节制铺开。

整改级别：**P0 信息结构重构**。

### 3. 历史记录 `/history`

结论：**部分达标**。

原型特征：

- 左侧深色热榜栏，右侧统一发起记录流。
- 顶部有状态筛选和平台筛选。
- 记录卡带平台徽标、状态、拍数、时间、社交指标和多入口按钮。
- 时间线竖线让历史记录有“发起时间线”的节奏。

当前偏离：

- 当前热榜在左侧，但不是深色侧栏，视觉权重和原型不同。
- 缺少筛选条，历史记录按身份分组，和原型的“统一发起流”不同。
- 记录卡按钮和平台入口能用，但排布更像功能列表。
- 数据少时首屏空白感明显，页面没有原型中的时间线节奏。

没照做的正当理由：

- 按身份分组能承接 P7/P14 身份能力，但与 P13 统一历史原型冲突；需要设计确认是否保留。若保留，应重新出终板原型，而不是混用现有原型验收。

整改级别：**P1 结构收敛**。

### 4. 多平台现场 `/world/:id/live`

结论：**未达高保真**。

原型特征：

- 左侧深色个人视角栏，中间微博与 Reddit 两个平台列并行。
- 两列之间有桥接路径和桥接数量。
- 右侧是跨平台桥接洞察、最新桥接和热词桥接。
- 顶部状态强调“本次发起 · 2 平台 · 发酵中 · 第 N 拍”。
- 微博与 Reddit 使用不同平台皮肤，不只是同一种卡片换标题。

当前偏离：

- 最新截图仍可能只有单个平台列，深色舞台右侧大面积空白或占位，首屏不像终板。
- 平台 icon 使用首字/文字替代，微博/Reddit 品牌识别弱。
- 桥接区域和右侧洞察未形成原型中的主叙事。
- 长内容堆叠、窗口策略和平台列滚动行为仍需要复核，避免再次出现无限拉伸。

没照做的正当理由：

- 真实 run 只有一个平台时不能伪造 Reddit 数据；但 UI 应进入单平台现场或给出“另一个平台未产生事件”的原型化空态。
- 技术难点在数据源和 SSE 事件聚合，但不解释 icon、容器比例和洞察栏缺位。

整改级别：**P0 多平台视觉与空态重构**。

### 5. Launch 复盘 `/world/:id/retro?launch=...`

结论：**未达高保真**。

原型特征：

- 左侧是本次发起上下文，主区是发酵时间线，右侧是创作者流量洞察。
- 中间每一波都有代表评论、参与用户、柱状节奏和情绪标签。
- 右侧建议持久化后以卡片形式展示，支持重新生成。
- 页面语义是“创作者复盘”，不是通用分析仪表盘。

当前偏离：

- 当前顶部先给一张摘要卡，然后进入分析 tab，信息结构与原型不同。
- 右侧桥接摘要/平台风味和主时间线分离，缺少原型中的三段式连续阅读。
- 主时间线密度不足，像指标面板，不像发酵复盘。
- 部分早期原型里的 debug/terminal 块已被认为不应复现，但当前实现也没有替换成等价的内容摘要模块。

没照做的正当理由：

- 去掉 debug/terminal 视觉是正确的设计修正；但应替换为平台角标、发起摘要或内容摘要，而不是完全改变页面骨架。

整改级别：**P1 复盘信息架构重排**。

### 6. 身份页 `/identity/:id`

结论：**未达高保真**。

原型特征：

- 头部是身份到达态：大头像、名称、圈层标签、来源世界与平台。
- 首屏信息卡包括影响力评分、粉丝估算、立场概览和平台账号。
- 下方有最近发言、参与过的世界、立场变化、被引用评论。
- 页面表达“这个人/身份的社交资产”，不是单次发起详情。

当前偏离：

- 当前是暗色个人卡 + 简单时间线 + 空曲线，信息维度少很多。
- 缺少平台账号、参与世界、最近发言、被引用评论等核心模块。
- 身份名、头像和影响力展示可读，但还没形成原型的身份资产面板。

没照做的正当理由：

- 部分模块需要后端聚合更多数据，但 P12/P14 已有 persons/worlds/events 读模型基础；至少可以先用已有数据支撑最近发言和参与世界。

整改级别：**P1 数据展示补齐 + 首屏重构**。

### 7. 单平台评论区 `/run/:id/live`

结论：**部分达标**。

原型/早期高保真特征：

- 页面像微博正文和评论区，用户第一眼能理解为社交媒体。
- 左侧个人视角栏、中心正文/评论、右侧通知栏、底部状态条各司其职。
- 回放态不出现无用过程控制。

当前偏离：

- 微博心智基本成立，但左侧深色栏和底部控制条仍偏“系统控制器”。
- 图标、头像、通知和操作栏有多个手写 SVG/文字符号，风格不统一。
- 部分布局在长评论时仍需滚动窗口策略，避免容器无限撑开或内部滚动与页面滚动打架。

没照做的正当理由：

- 左栏可保留，因为它表达“第一人称视角”；但底部控制条在回放/完成态应减重或只保留必要操作。

整改级别：**P2 图标统一 + 控制条减重**。

### 8. 选圈子 / 旧 Gallery

结论：**待重新定性**。

现状：

- P13/P14 后顶层信息架构已转为“发起 / 世界 / 历史”，旧“选圈子”不再是终板主入口。
- 如果该页仍保留，应作为发起页里的受众选择组件，而不是独立旧页面继续验收。

整改级别：**P2 产品范围确认**。

## 横向设计问题

### 图标与 iconfont

当前问题：

- 顶栏品牌、通知、世界、发布、身份、轮次、平台标识、桥接等图标来源不统一。
- 多处使用文字符号或首字母代替图标，如 `◎`、`⌁`、`◈`、`◌`、`→`、`‹`、`...`。
- 平台卡没有对齐原型里的微博 / Reddit / 多平台图标。

整改建议：

- 新建本地 SVG icon 系统，不新增外部依赖。
- 所有页面统一使用同一套线性图标、统一 stroke、尺寸、对齐和 selected 状态。
- 平台 icon 单独做品牌化近似图形，避免首字母圆点。

### 容器与间距

当前问题：

- 发起页当前最大宽度、左右栏比例、顶部/卡片间距与原型不一致。
- 世界页和历史页的主区域过窄或过长，页面节奏不像原型。
- 背景色和卡片阴影有时更像后台管理页，而不是终板里的轻量社交产品。

整改建议：

- 桌面统一使用终板原型的 `max-width`、左右栏宽度和 card radius。
- 首页级页面先对齐首屏，再处理长列表状态。
- 任何新增功能若破坏首屏比例，必须折叠到二级入口。

### 真实数据与原型样例

当前问题：

- 真实数据中的空世界、单平台 run、少量记录会让页面观感差。
- 但这些数据问题不能解释组件层级没有按原型实现。

整改建议：

- 分开两类验收：组件高保真 fixture 和真实数据健壮性。
- fixture 用来锁定原型视觉；真实数据用来验证空态、长列表、单平台、缺平台和性能。

## 2026-07-07 世界/历史热榜栏回填

用户验收发现 `/worlds` 与 `/history` 左侧区域空白，之前误判为应隐藏空栏；复核 P14 原型后确认这是错误方向。原型中的热榜是桌面信息架构的一部分，应稳定存在，问题应从数据供给链路排查。

根因：

- `TrendRail` 在没有有效 topic 时直接 `return null`，导致左侧网格列仍占位但没有内容。
- `/worlds` 的热榜数据只使用 `latestRunContent`；当世界已有名称、身份和运行记录，但 latest content 为空时，热榜没有 fallback。
- `/history` 的主列表已经按 `launches` 分组展示，但热榜仍只读取 `runs`；多平台历史只有 launch 记录或 run summary 暂缺时，列表有内容而热榜为空。

整改：

## 2026-07-07 多平台现场滚动容器回填

用户验收发现 `/world/:id/live` 多平台现场仍由内容自然撑开，长评论会把整页高度无限拉长；这与 P13 多平台现场“固定场景容器 + 平台现场内部滚动”的原型心智不一致。

根因：

- `world-live-stage` 是普通文档流容器，没有 viewport 高度边界。
- 微博/Reddit 列只在 `platform-scroll-viewport` 上设置 `max-h-[760px]`，列会滚动，但外层 stage 和桥接栏仍随整体内容参与页面撑高。

整改：

- `world-live-stage` 改为 `h-[calc(100vh-7rem)] max-h-[calc(100vh-7rem)] overflow-hidden flex flex-col`。
- 新增 `world-live-content` 内容区，使用 `min-h-0 flex-1 overflow-hidden`。
- 微博、Reddit、缺平台空态和桥接路径都改为 `min-h-0` 内部滚动模型，平台现场使用 `flex-1 overflow-y-auto`，移除固定 `max-h-[760px]`。
