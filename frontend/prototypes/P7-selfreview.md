## 逐屏核对

### 发起页

- 心智词表：未出现 `agent/OASIS/仿真/工作台/trace/simulation/dashboard`。
- 三寄存器：深墨围观壳、近白社交输入区、暖琥珀身份主色边界清晰。
- 关键组件：内容输入、persona 三卡、身份选择、圈子、轮次、成本预估条均出现。
- 触达：移动端按钮与卡片面积满足 44px 触达。
- 实现注意：导航按计划使用 `发起 / 世界 / 历史`；图中侧栏扩展项不作为 P7 必交。

### 历史按身份

- 心智词表：未出现禁用词。
- 数据来源要求：身份卡、粉丝、影响力、run 节点必须由 `PersonView` + run summary 派生，禁止前端造假。
- 三寄存器：历史列表为社交内容区，热榜/控制为围观壳；按钮和高亮使用暖琥珀/冷靛蓝体系。
- 业务含义：按身份分组清晰，组内 run 节点可承载评论、转发、点赞、影响力变化。
- 实现注意：图中热榜是布局参考，内容必须来自保存的历史/投影数据。

### 身份/世界主页

- 心智词表：未出现禁用词；图中 `Run #` 属原型英文占位，实现时替换为 `第 N 次围观` 或 `历次围观`。
- 三寄存器：世界层使用深墨背景、暖琥珀身份/影响力主色、冷靛蓝连接，未使用图表库默认配色。
- 关键组件：身份卡、总影响力、粉丝、立场漂移时间线、影响力曲线、名下账户均出现。
- 移动端：信息层级保留，账户卡片与曲线在移动端可读。
- 实现注意：平台账户列表按后端 `Account[]` 渲染；无账户时显示诚实空态。

## 实现约束

- 可见文案继续避开禁用心智词。
- 世界层 token 使用 `world` 命名空间，身份主色与 `brand #E8A13A` 一致。
- 所有累积数据从 `PersonView` 派生，不在前端维护本地身份增长状态。
- 图表/曲线采用自定义 CSS/SVG，禁用图表库默认配色。
