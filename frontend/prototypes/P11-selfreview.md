# P11 世界入口原型自审

原型图：`frontend/prototypes/world-entry.png`

## 覆盖范围

- 发起页：平台多选，选择微博 + Reddit 后出现“多平台并发”提示，并保留单平台围观模式入口。
- 世界总览：展示持续身份、平台现场、影响力、整体情绪和“看现场”入口。
- 多平台现场：微博现场与 Reddit 现场并列，使用桥接路径表达跨平台发酵。
- 移动端：覆盖发起、总览、历史、现场和建议区域的窄屏布局。

## 自审结果

- 心智词表：未使用 `agent`、`OASIS`、`仿真`、`工作台`、`后端`、`配置中心`、`世界地图` 等世界层禁用词；界面表达使用“世界 / 平台 / 现场 / 发酵 / 桥接”。
- 配色：世界层语义色对齐 `frontend/src/design/tokens.ts`：深色壳层使用 `world.surface`，身份与主行动使用 `world.identity`，桥接线使用 `world.line`，影响趋势使用 `world.influenceUp` / `world.influenceDown`。实现阶段不得硬编码 Tailwind 默认色阶承载语义。
- 发起路径：多平台选择与“多平台并发”提示清晰，用户能理解同一条内容会在多个平台同步发酵。
- 世界总览：卡片能看出身份、平台覆盖、影响力和现场入口，不引入“后台管理”心智。
- 多平台现场：微博与 Reddit 两列并列，桥接路径放在中间，能表达内容从一个平台外溢到另一个平台。
- 组件取舍：原型为视觉方向，生产 UI 文案和数据必须代码原生渲染，不能把截图当 UI 使用。

## 实现约束

- `ComposeScreen` 只在选择两个及以上平台时走多平台发起；单平台路径继续使用原 `createRun`。
- `MultiPlatformLiveScreen` 保持可注入 `events` 的测试能力，同时路由态从 `/api/worlds/{id}/events` 拉真实事件。
- “世界”导航进入 `/worlds`，不再进入旧 gallery。
- 世界总览和现场的新 UI 若需要情绪、影响力、桥接等语义色，只能从 `design/tokens.ts` 引用。
